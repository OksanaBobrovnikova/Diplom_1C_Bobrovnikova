#  Инструкция по начальной настройке нового функционала

Данная инструкция предназначена для специалистов заказчика, ответственных за настройку и администрирование информационной базы. Она описывает действия, которые необходимо выполнить в существующей информационной базе.

**Важно:** Перед началом настройки рекомендуется сделать резервную копию информационной базы!

## Шаг 1: Обновите конфигурацию
Обновите конфигурацию базы данных до версии, содержащей разработанную функциональность. Это можно сделать с помощью стандартных средств платформы "1С:Предприятие":
1. Запустить конфигуратор;
2. Выбрать "Конфигурация" далее "Сравнить, объединить с конфигурацией из файла...";
3. Укажите файл cf содержащий новую конфигурацию;
4. Ознакомьтесь с результатом сравнения и объединения. Обратить особое внимание на возможные конфликты;
5. Принять изменения;
6. Обновить конфигурацию базы данных.

## Шаг 2: Создание пользователей
1. В подситеме "Администрирование" далее "Настройки пользователей и прав" далее "Пользователи".
2. На вкладке "Пользователи" создать необходимых пользователей:
   * Администратор с правами доступа Администратор (выбрать соответствующий профиль из списка на вкладке "Права доступа");
   * Бухгалтер ИТ Фирмы с соответствующими правами доступа (выбрать профиль из списка на вкладке "Права доступа");
   * Кадровик-расчетчик с соответствующими правами доступа (выбрать профиль из списка на вкладке "Права доступа");
   * Специалист с соответствующими правами доступа (выбрать профиль из списка на вкладке "Права доступа");
   * Менеджер с соответствующими правами доступа (выбрать профиль из списка на вкладке "Права доступа").


## Шаг 3: В подсистеме "Обслуживание клиентов" настроить Телеграм-бот.
1. Создание бота:
   * Напишете в Телеграм [https://t.me/BotFather](https://t.me/BotFather) команду "/start"
   * Напишите команду "/newbot"
   * Выберите имя вашего бота, которое будут видеть пользователи
   * Выберите идентификатор вашего бота (он должен заканчиваться на "bot")
   * Скопируйте токен, который пришлет BotFather и сохраните его в константу
2. Создание группы:
   * Создайте группу в Телеграм
   * Добавьте в группу только что созданного бота
   * Назначьте боту права администратора
   Получить идентификатор группы:
   * Напишите любое сообщение в группу
   * С помощью браузера или Postman выполните запрос `https://api.telegram.org/bot[ВашТокен]/getUpdates`. В полученном JSON найдите идентификатор группы, в которой получено сообщение, и сохраните его в константу.
3. Ностроить регламентное задание по отправке сообщений телеграм-боту

## Шаг 4: Заполнить константы:
1. "Идентификатор группы оповещения" - заполняется идентификатором группы Телеграмм (см. Шаг 3).
2. "Токен управления Телеграм-ботом" - заполняется токеном бота Телеграмм (см. Шаг 3).
3. "Номенклатура абонентская плата" - заполняется из справочника "Номенклатура" соответствующим по смыслу наименованием, при необходимости создайте нужную номенклатуру.
4. "Номенклатура работы специалиста" - заполняется из справочника "Номенклатура" соответствующим по смыслу наименованием, при необходимости создайте нужную номенклатуру.

## Шаг 5: Перед первым использованием подсистемы "Расчет зарплаты"
1. Заполнить справочник графики работ (например Пятидневка, шестидневка). 
2. В разделе "Сервис" с помощью обработки "Заполнение графика", заполните соответствующий график. В поле обработку Выходные дни указываются дни недели цифрами (подряд без пробелов), которые являются выходными. Например для графика пятидневка поле заполняется значением "6,7".

## Шаг 6: Заполнить данные о сотрудниках в подсистеме "Расчет зарплаты"
1. Кадровик-расчетчик в обязательном порядке должен заполнить в разделе "Регистры" сведения Условия оплаты сотрудников.
2. В разделе "Документы" заполнить:
   * график отпусков;
   * начисления заработной платы;
   * начисления фиксированных премий;
